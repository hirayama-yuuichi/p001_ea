## そもそも強化学習ってなに？

強化学習とは、AIが「試行錯誤」を繰り返しながら、もっとも良い行動を自分で学んでいく仕組みです。

イメージとしてはゲームの攻略と同じです。最初はうまくいかなくても、何度も繰り返すうちにコツをつかんでいきます。FXの場合は「買う・売る・何もしない」の3択を選び続けながら、「儲かったら褒める・損したら罰する」という形で学習させます。

---

## 1. どんな戦略（アルゴリズム）を使うか

### DQN（ディーキューエン）
正式名称: Deep Q-Network

**どういうもの？**  
「今の状況でどの行動をすれば、将来いちばん儲かるか」を予測するニューラルネット（脳のようなもの）です。チェスや囲碁のAIにも使われている考え方と同じ系統です。

**良いところ**
- 実装例がいちばん多く、参考資料を探しやすい
- 仕組みが比較的わかりやすい

**気をつけること**
- スプレッド（取引コスト）を考えると、ただ持ち続けるだけの戦略に落ち着いてしまいやすい
- 相場の雰囲気が変わると、学習内容が古くなってしまう

**結論**: まず最初に試すアルゴリズムとしておすすめ

---

### PPO（ピーピーオー）
正式名称: Proximal Policy Optimization

**どういうもの？**  
DQNと同じく強化学習の手法ですが、「行動の方針（ポリシー）」を少しずつ安定して更新していく仕組みです。急激な方針変更をしないように制約をかけるのが特徴です。

**良いところ**
- DQNより学習が安定しやすい
- Stable-Baselines3（後述）というライブラリで一番使いやすく実装できる

**気をつけること**
- チューニング（調整）が難しく、パラメータ設定を間違えると高頻度売買に陥って損失を積み上げることがある

**結論**: DQNと並行して試してみる価値あり

---

### Transformer（トランスフォーマー）＋ Attention（アテンション）

**どういうもの？**  
ChatGPTなどの大規模言語モデルでも使われている仕組みです。「過去のどの時点が今の判断に重要か」を自動で見つける能力（Attention）があります。

FXでいうと、「3時間前の急落が今のエントリー判断に関係しているかどうか」を自動で気づくようなイメージです。

**良いところ**
- 最新の研究では最も性能が高い（Sharpe比で3〜6%改善の報告あり）
- 複数の時間軸を同時に考慮できる

**気をつけること**
- 実装が難しく、学習に必要なデータ量も多い
- 日本語の実装例がまだ少ない

**結論**: 最終的に目指す形。最初はDQN/PPOで経験を積んでからがいい

---

### どの順番で進めるか

```
① まず PPO か DQN で動かしてみる（シンプルに始める）
② 両方比較してみる
③ 性能が上がったら Transformer を特徴量の抽出に使う形で組み込む
```

## 2. どうやって決済するか

### 固定時間決済（5分後に必ず売る）

**どういうもの？**  
エントリー（ポジションを持つこと）から5分後に、利益・損失問わず必ず決済する方法です。

**良いところ**
- 仕組みが単純で、バックテスト（過去データでの検証）がやりやすい
- ポジションを長く持つリスクがない

**気をつけること**
- 上昇トレンドが続いていても5分で切ってしまうため、利益を取り逃がす
- 逆に損失が出ていても強制決済になる

**結論**: まずはこれで始めるのがいい。シンプルな設計のほうがバグも見つけやすい

---

### TP/SL（利確・損切り）

**どういうもの？**  
- TP（テイクプロフィット）: 「〇〇pips 上がったら利確する」という上限
- SL（ストップロス）: 「〇〇pips 下がったら損切りする」という下限

pipsとは: FXの値動きの最小単位。ドル円なら0.01円 = 1pip。

**5分足FXの場合の目安**
- TP: 5〜15pips
- SL: 5〜10pips
- リスクリワード比（TP÷SL）: 1:1〜1:2 が標準的

**おすすめの設定方法**  
ATR（平均的な値動きの幅）の0.5〜1.5倍をTP/SLにする「動的設定」が安定しやすいと言われています。相場の荒れ具合に応じて自動で調整されるイメージです。

---

### AIに決済も判断させる

エントリーと同様に、決済タイミングもAIが「Hold（持ち続ける）」「Close（決済する）」と判断する設計です。

**ポイント**
- Buy / Sell / Hold の3択をAIが選ぶ形にする
- 報酬（褒美）の設計に工夫が必要：
  - スプレッド（取引コスト）を毎ステップ引く
  - 単純な損益より「Sharpe比（リターンの安定性）」を使うほうが安定した学習になる

## 3. バックテストとMT5との連携

### バックテストとは？

過去の価格データを使って「この戦略は過去だったら儲かっていたか？」を検証することです。未来のことはわからないので、まず過去で検証するのが基本です。

---

### MT5（MetaTrader 5）とPythonをつなぐ方法

MT5はFXのチャートや注文を管理するソフトです。PythonでAIを作り、MT5と連携させることで自動売買ができます。

**基本的な流れ**

```
Python（AI・学習コード）← データ取得 →  MT5（チャート・注文）
                        ← 注文送信 →
```

**主な連携パターン**

| 方法 | 内容 | おすすめ度 |
|---|---|---|
| MT5 Pythonライブラリ（直接） | pip一発でデータ取得・注文が両方できる | ★★★ まずはこれ |
| gym-mtsim | MT5をAIの学習環境として使えるラップ | ★★★ MT5連携まで踏み込むなら |
| ONNX経由でEAに組み込む | 学習済みAIをMT5のEA（自動売買スクリプト）に埋め込む | ★★ 本番運用時 |

**注意**: MT5 Pythonライブラリは **Windowsでしか動かない** ので、EC2のWindows環境で実行します。

---

## 4. どのライブラリを使うか

### Stable-Baselines3（SB3）
強化学習のアルゴリズム（PPO・DQN・A2Cなど）をまとめて使えるPythonライブラリです。設定が統一されているので、アルゴリズムの切り替えがしやすいです。現在も活発に開発が続いていて信頼性が高い。

### gym-anytrading
FXや株のトレードを強化学習で試すための「練習環境」を提供してくれるライブラリです。自分でCSVデータを用意すれば、MT5がなくてもバックテストできます。まず動かしてみたいときに最適。

### gym-mtsim
gym-anytradingの兄弟のようなライブラリで、MT5から直接データを取得してAIを学習させられます。バックテスト → MT5連携 を一体でできるのが強みです。

### FinRL
研究用に比較実験がしやすい大きなフレームワーク。論文の実装例が多いですが、依存ライブラリが重いので最初は後回しで良いです。

## 5. 全体の進め方（おすすめの順番）

```
【段階1】 とにかく動かす
  → gym-anytrading + SB3(PPO) + CSVデータ（ローカルLinux）
  → まず「動く状態」を作ることが大事

【段階2】 MT5とつなぐ
  → gym-mtsim + SB3 + EC2 Windows MT5
  → リアルなデータで学習・バックテスト

【段階3】 本番運用
  → 学習済みモデル → ONNX形式 → MT5 EAに埋め込む
  → または Python から直接注文を送る
```

## 6. 全体で気をつけること

**スプレッドは必ず含める**  
スプレッド（買値と売値の差）は5分足では往復2〜3pipsになることがあり、これを無視すると「バックテストでは儲かるのに本番では損する」という事態になります。研究でも、手数料なし → 手数料ありで多くのAIが赤字転落した例が報告されています。

**データは訓練・検証・テストに分ける**  
同じデータで学習も評価もすると、「暗記しただけで応用が効かないAI」が出来上がります。一般的には 7:1:2（訓練:検証:テスト）で分けます。

**過学習（オーバーフィッティング）に注意**  
過去データに最適化しすぎると、未来の相場には通用しません。ウォークフォワード検証（時系列を前から順にテスト）が理想的です。

**EC2の4時間シャットダウンに注意**  
EA（自動売買）が動いている間はシャットダウンされないよう、AutoShutdownタスクを無効にするか、保有ポジションがない時間帯に設定する必要があります。

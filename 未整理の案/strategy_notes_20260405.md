# 戦略検討メモ 2026-04-05

## 1. 戦略：深層強化学習 × マルチタイムフレーム × 指標発表

### 現在の方針
- マルチタイムフレーム（5分足をベースに1時間足・日足の特徴量を混在）
- 指標発表イベントは別扱い（フィルタリングまたは特徴量として追加）
- 深層強化学習（DQN or Transformer系）

### 他の方法とメリデメ

| 手法 | メリット | デメリット |
|---|---|---|
| **DQN（現行候補）** | 実装例多い、入門しやすい | 過学習しやすい、相場変化に弱い |
| **DQN + LSTM** | 時系列の記憶を保持 | 学習不安定になりやすい |
| **Transformer / Attention** | 長期依存を自動把握、最新研究主流 | 実装難度高い、学習データ大量必要 |
| **PPO / A3C（方策勾配系）** | DQNより安定した学習 | チューニング難しい |
| **ルールベース（MAクロスなど）** | 解釈しやすい、実装簡単 | 相場変化に対応できない |
| **アンサンブル（ルール＋RL）** | 堅牢性が上がる | 複雑になる |

### 推奨方針
**DQN + マルチタイムフレーム で始めて、Transformer に移行** が現実的。
mhtmlの調査でも「DQN単体 → ほぼ苦戦 → Transformer移行」の流れが確認されている。

## 2. 売買手法：5分後に必ず売る（固定時間決済）

### メリット
- 実装がシンプル
- ポジション保有リスクが限定される
- MT5のEA実装が楽

### デメリット
- トレンドが続いているときに早く切りすぎる
- 逆に損失が確定していても5分で強制決済される

### 他の決済方法

| 手法 | メリット | デメリット |
|---|---|---|
| **固定時間決済（5分）** | シンプル、実装容易 | 利益を取り逃がす |
| **TP/SL（利確・損切り）** | リスク管理しやすい | 値の設定が難しい |
| **RL が決済も判断** | 最適な決済タイミングを学習 | 状態空間が複雑になる、学習困難 |
| **トレーリングストップ** | 利益を伸ばせる | 実装やや複雑 |

### 推奨方針
**まずは固定時間（5分）で始める**。シンプルな設計でバックテストを回し、
性能が上がったら TP/SL を追加するのが良い順序。

## 3. バックテスト：MT5との同期

### 課題
- Pythonで学習したモデルの結果と、MT5上でのEA稼働結果を一致させたい

### 方法

**A. Python側でMT5と同じロジックを再現（推奨）**
- `MetaTrader5` Python ライブラリで MT5 から直接データ取得
- Python で同じ特徴量・エントリーロジックを実装してバックテスト
- MT5のストラテジーテスターは「参考値」として使う

```python
import MetaTrader5 as mt5
mt5.initialize()
rates = mt5.copy_rates_from_pos("USDJPY", mt5.TIMEFRAME_M5, 0, 1000)
```

**B. MT5 ストラテジーテスター ＋ Python モデルを連携**
- EAからPythonモデルを呼び出す（DLL経由 or ZMQ通信）
- 実装は重いが最も正確

**C. MT5のティックデータをエクスポートしてPythonで処理**
- MT5から `.csv` でデータ出力 → Python でバックテスト
- 簡単だがスプレッドやスリッページが再現できない

### 推奨方針
**まず A（MetaTrader5ライブラリ）** で始める。
Python側でバックテストを完結させ、良い戦略ができたらEAに移植。

## 4. 全体の考慮点

### データ
- 最低でも2〜3年分の5分足データが必要
- 訓練・検証・テスト期間を分ける（例: 7:1:2）
- 指標発表時刻のデータは別管理（除外 or 特徴量）

### 学習
- 過学習に注意：テスト期間は訓練に使わない
- ウォークフォワード検証が理想（時系列を前から順にテスト）

### コスト
- スプレッドを必ず考慮（5分足で往復2〜3pips = 大きい）
- スリッページも本番では発生する

### 運用
- EC2は4時間自動シャットダウン → EA稼働中は注意（停止しないよう設定変更が必要）
- モデルの再学習タイミングをどうするか（週次？月次？）

### 段階的進め方
1. データ収集（MetaTrader5ライブラリ）
2. 特徴量設計（MA、RSI、ATR + 時間足マルチ）
3. DQNでシンプルなバックテスト
4. 評価指標を決める（Sharpe比、最大ドローダウン）
5. 改善（LSTM追加、Transformer移行）
6. MT5 EA として実装・デモ運用
